# cub3D Project - User Stories and Tasks

## Epic: 3D Ray-Casting Maze Game

### User Story 1: Window Management
**As a** player  
**I want** a responsive game window  
**So that** I can have a smooth gaming experience

#### Tasks:
- [x] Initialize MiniLibX library
- [x] Create game window with configurable dimensions
- [x] Implement window close functionality via ESC key
- [x] Implement window close functionality via red cross button
- [ ] Handle window minimize/maximize events
- [ ] Ensure smooth window switching between applications
- [ ] Test window responsiveness under different scenarios

---

### User Story 2: Scene Configuration File Parser
**As a** player  
**I want** to load game configurations from a .cub file  
**So that** I can play different maze layouts

#### Tasks:
- [x] Create file validation to check for .cub extension
- [x] Implement file opening and reading functionality
- [x] Parse texture paths (NO, SO, WE, EA identifiers)
- [x] Parse floor color (F identifier with RGB values)
- [x] Parse ceiling color (C identifier with RGB values)
- [x] Validate RGB values are within [0,255] range
- [x] Handle multiple spaces between elements
- [x] Handle empty lines between elements (except in map)
- [x] Ensure map is always the last element in file
- [x] Create error handling for file read failures

---

### User Story 3: Map Parser and Validator
**As a** game developer  
**I want** a robust map parser  
**So that** the game loads valid maze layouts

#### Tasks:
- [x] Parse map characters (0, 1, N, S, E, W)
- [x] Validate map contains only allowed characters
- [x] Identify player starting position (N, S, E, W)
- [x] Determine player initial orientation based on spawn character
- [x] Preserve spaces within the map structure
- [x] Validate map is completely enclosed by walls
- [x] Implement flood-fill algorithm to check map closure
- [x] Handle irregular map shapes
- [x] Store map in appropriate data structure (2D array/list)
- [x] Create error messages for invalid maps

- [x] Load West wall texture from specified path

### User Story 5: Ray-Casting Engine
**As a** player  
**I want** a 3D perspective view of the maze  
**So that** I experience immersive first-person gameplay

#### Tasks:
- [ ] Implement ray-casting algorithm
- [ ] Calculate ray directions for each screen column
- [ ] Perform DDA (Digital Differential Analysis) for ray traversal
- [ ] Detect wall collisions
- [ ] Calculate perpendicular distance to walls
- [ ] Determine wall slice height based on distance
- [ ] Identify which wall face was hit (N, S, E, W)
- [ ] Optimize ray-casting for performance
- [ ] Handle edge cases (corners, boundaries)
